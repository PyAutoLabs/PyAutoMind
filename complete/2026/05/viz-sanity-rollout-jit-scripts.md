- notes: Phase D.1 of z_features/fast_visualization.md. Rolls the __Visualization Sanity__ pattern from PR #111 (imaging pilot) out to the remaining JIT-cached visualization scripts in both autolens_workspace_test (4 scripts: imaging delaunay + rectangular + interferometer + point_source) and autogalaxy_workspace_test (2 scripts: imaging + interferometer). 6 scripts total, ~280 lines added. Each block uses a deterministic SIE sanity tracer for lensing-side assertions (1 tangential CC + finite positive einstein_radius + warm-call < 100 ms) plus dataset-specific add-ons: interferometer asserts fit.model_data (complex Visibilities) finite/non-zero, autogalaxy variants assert fit.model_data finite/non-zero + figure_of_merit finite (no Tracer → no lensing-side assertions), point_source omits a script-specific FoM assertion because the prior-median position can legitimately give chi²=-inf (positions outside image-pair basin). Validated 4/6 scripts end-to-end locally; the 2 pixelization variants (delaunay/rectangular) hit a pre-existing Part-1 caching-ratio assertion fragility on CPU + small datasets (cache speedup < 2x) that blocks local validation but doesn't affect the Sanity block syntax (identical to imaging pilot). Gotchas: (1) Initial attribute-name guess `fit.model_visibilities` was wrong — the actual attribute on FitInterferometer is `fit.model_data` returning an aa.Visibilities; np.abs().sum() works on both Array2D and Visibilities so the assertion is dataset-type-agnostic. (2) point_source script didn't `import numpy as np` at module top (only `jax.numpy as jnp`); added inside the Sanity block as a local import. (3) Same point.py pre-existing JAX vmap CI red on autolens_workspace_test — merged anyway, identical to two previous tasks. Phase D.2 remains: visualization_jax*.py scripts get Sanity blocks, plus authoring new modeling_visualization_jit.py for ellipse (autogalaxy), weak lensing (autolens), quantity (autogalaxy).
