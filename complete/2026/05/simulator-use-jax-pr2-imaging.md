- notes: PR 2 of 4 from simulator_use_jax.md. Shipped `aa.SimulatorImaging(use_jax=True)` constructor flag with `_xp` property and `via_image_from(xp=None)` auto-fallback. Poisson noise routes through `jax.random.PRNGKey(noise_seed) + jax.random.poisson` on the JAX path; `preprocess.poisson_noise_via_data_eps_from` and `preprocess.data_eps_with_poisson_noise_added` gained `xp=np` parameter. PyAutoLens `via_tracer_from` and PyAutoGalaxy `via_galaxies_from` overrides default `xp=None` and fall back to inherited `self._xp`. **Known limitation flagged in simulator docstring:** @jax.jit wrapping is blocked by pre-existing PyAutoArray Array2D.native jit-incompatibility (array_2d_via_indexes_from uses indexed assignment of native-index tuples). Eager JAX path works and is sufficient for batched simulations / parameter sweeps. Separate task needed to refactor slim/native reshape for full @jax.jit support. **Other notable decision:** the `if xp.isnan(noise_map.array).any():` runtime NaN guard now NumPy-only — Python `if <tracer>:` triggers TracerBoolConversionError under JAX; JAX users must validate their exposure_time / background_sky_level themselves. Test-state pollution observed between PyAutoLens and PyAutoGalaxy imaging tests when both repos run in sequence (2 imaging/model tests fail combined, pass standalone) — pre-existing, not caused by this PR. Tests: PyAutoArray 831/831, PyAutoLens 317/317, PyAutoGalaxy 918/918 standalone. Eager parity script (autolens_workspace_test/scripts/imaging/simulator_use_jax_parity.py) confirms numpy and JAX paths agree to atol=1e-8 for noise-free Sersic + Isothermal lens. **Remaining Phase 2 work:** PR 3 (SimulatorInterferometer.use_jax + via_image_from signature symmetry), PR 4 (xp=np + jnp-grid mismatch ValueError in AbstractMaker). Plus separate task for Array2D.native jit-safety refactor. Worktree removed; feature branches deleted local + origin.
