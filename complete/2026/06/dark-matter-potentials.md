- notes: Fixed "TypeError: MassProfile.potential_2d_from() got an unexpected keyword argument 'xp'" crashing tracer visualization for any model with an elliptical NFW. Root cause was NOT PyAutoFit/model-composition (user's initial suspicion) nor the N=0 fixed PowerLaw/shear — the elliptical dark-matter profiles (NFW, gNFW, gNFWSph + MCR/Virial variants) never implemented potential_2d_from and fell through to the abstract MassProfile base; recent JAX xp-threading on Galaxy.potential_2d_from's sum-lambda turned a latent NotImplementedError into the xp TypeError, and the recent autolens fits_tracer change (writes a `potential` FITS extension) was the first thing in the run loop to call it (likelihood path never touches the potential — hence "fit healthy, only viz crashes"). Fix: one potential_2d_from on gNFW via MGE decomposition (same sigma range as its deflections_2d_via_mge_from, so grad(psi)=alpha is self-consistent), inherited by NFW/gNFWSph/variants. Also fixed a pre-existing bug in NFWSph's analytic potential — prefactor was 2*r_s*kappa_s, should be 2*r_s^2*kappa_s (functional form ln^2(x/2)-arctanh^2(sqrt(1-x^2)) was correct); grad(psi) was exactly 1/r_s too small (NFWSph was FAILing the dark.py grad(psi)=alpha check on main). Threaded xp/**kwargs into base MassProfile.potential_2d_from so genuinely-unimplemented profiles raise a clean NotImplementedError. Validation: dark.py self-consistency flips NFW/gNFW/gNFWSph SKIP->PASS and NFWSph FAIL->PASS (grad med ~8e-4); MGE potential matches corrected NFWSph analytic to 1.5e-5; 933 PyAutoGalaxy tests pass; end-to-end fits_tracer smoke writes finite nonzero POTENTIAL extension for the user's PowerLaw+shear+NFW model. Two pre-existing follow-ups filed in planned.md: (1) NFWTruncatedSph potential fails grad(psi)=alpha (MGE sigma range radii_max=truncation_radius*5 too narrow); (2) PIEMass has no potential and no MGE/CSE hook (needs analytic Kassiola&Kovner port or dPIEMass r_s->inf limit). Gave Jacob a manual site-packages hotfix (make base MassProfile.potential_2d_from accept xp and return zeros) to unblock until release.
